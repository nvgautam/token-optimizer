## Key decisions

- Oracle owns milestone planning (execution_plan.md); orchestrator owns execution — no re-derivation of plan at runtime
- Parallelism rounds (which tasks CAN run in parallel) are oracle-defined; grouping within rounds (which tasks SHOULD share an agent) is orchestrator-computed at runtime based on shared reads overlap
- Grouping across round boundaries is explicitly forbidden — rounds are dependency gates
- CLAUDE.md is loaded by all sessions (human + oracle + orchestrator); human-focus notes must not live there
