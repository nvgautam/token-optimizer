## Key decisions
- PTY shell is stdlib-only, zero LLM — fully deterministic; compiled via Nuitka for IP protection
- Symbol index never in project tree — ~/.agentflow/cache/ — users never see it, protects methodology IP
- Oracle generates tasks.json upfront (all tasks decomposed); orchestrator extends via lazy milestone decomposition
- No separate handoff files: architecture.md UNRESOLVED items = oracle resume signal; execution_plan.md incomplete milestones = orchestrator resume signal
- Token savings come from skills + living state documents (not PTY) — PTY automates the trigger and protects IP
- Hybrid handoff threshold: 40K absolute OR 30% of window — whichever fires first; handles 1M-window models correctly
- Orchestrator = Staff Engineering Lead (execute), Oracle = Senior PE + PM + Designer (design + prioritize)
- PTY sequenced last in v1; user manually triggers /handoff until then
- Tier/licensing deferred; naming/branding deferred; Codex provider v2; brownfield refactoring v2
